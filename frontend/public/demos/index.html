🔍 What Gets Scraped?

The Sales Agent extracts the following from each .rfp-listing element:

  • RFP Title: From .rfp-title
  • Client Name: From .rfp-client or meta section
  • Deadline: Parsed from date strings
  • Project Value: Extracted from value field
  • Description: From .rfp-description
  • Specifications: From .rfp-specs or spec tags
  • Client Type: Inferred from client name (government/private)

📋 Testing Workflow

  1. Discover RFPs: Sales Agent scrapes the portal page
  2. Apply 90-Day Filter: RFPs due beyond 90 days are filtered out
  3. Calculate Go/No-Go Score: Based on urgency, value, and fit
  4. Create Tickets: Qualified RFPs stored in rfp_tickets table
  5. Queue for Processing: Pushed to Redis queue for Technical & Pricing agents

🛠️ API Testing

Test the Sales Agent API directly:

curl -X POST http://localhost:8005/api/sales/intake-url \
-H "Content-Type: application/json" \
-d '{"url": "http://localhost:5173/demos/test-simple.html"}'

Or use Python:

from agents.sales.agent import SalesAgent

agent = SalesAgent()
rfps = agent.discover_rfps_from_url(
"http://localhost:5173/demos/test-simple.html"
)
print(f"Found {len(rfps)} RFPs")
